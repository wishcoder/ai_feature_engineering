Issue Summary

Issue ID: SVB003

Component: Service B (Java)

Issue Type: Unavailability

Date Reported: 2024-03-26

Affected Systems: Service B

Issue Description

Service B is currently not responding, leading to a breakdown in the service chain affecting downstream database access.

Architecture Overview

Outside network connects to Router, Router connects to firewall that sends traffic to App, App connects to Service A, which connects to Service B, which then connects to Service C (Database).

Possible Causes

  • Service B process crash due to memory leaks.
  • Network partitioning preventing access to Service B.

Resolution Steps

  1. Restart Service B: Attempt to restart the service to recover from any non-persistent issues.
  2. Check Logs: Review Service B's logs for errors that occurred just before the unavailability.
  3. Network Diagnostics: Run network diagnostics to ensure there is no partitioning issue.

Resolution Outcome

Restarting Service B and applying a patch to fix the memory leak issue restored its availability.
